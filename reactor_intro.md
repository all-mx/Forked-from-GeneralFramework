# 简介
## 应用
本项目与大多数嵌入式框架一样分为 `Bsp`、`Module`、`Sys`、`App`层
用户 ***无需更改*** App层以下的任何代码，只需要通过调用 / 编写 `Application` 即可实现逻辑

## 基本运行逻辑
由system接管所有的 `RTOS` 线程
有四大基本线程，其中应用在注册后，以200Hz运行在 `Application` 线程中
用户只需要在 `MainFrame` 里调用 `System.RegistApp()` 即可注册激活应用;
在注册时，会触发应用的 `Start` 函数；运行时，则会循环调用 `Update` 函数

## 架构隔离
整个 `Mod` 及以上的框架，都不会接触到HAL库。这是由于隔离设计实现的。`BSP` 由于天生面向硬件，
其在cpp中会引入HAL库，但是在hpp中绝不会；这防止了上层的导入泄露

## 控制量单位规范

为避免跨层参数误配，控制相关模块统一遵循以下单位约定：

- 角速度：用户接口可用 `RPM`，内部控制计算统一使用 `rad/s`。
- 角位置：内部与上层目标统一使用 `rad`。
- 电流：控制器输出物理量使用 `A`，总线发送时转换为电流编码值 `code`。
- 带宽参数：用户配置常用 `Hz`，控制器内部可转换为 `rad/s`。

常用换算：

- `rad/s = rpm * 2pi / 60`
- `rad = angle_code / 8192 * 2pi`
- `current_code = I(A) * 16384 / 20`

在 `motor_dji` 模块中，速度接口 `SetSpeed()` 使用 RPM，位置接口 `SetPos()` 使用 rad，
`_sloperate` 使用 A/s 语义配置，内部在发包前进行编码域换算。
